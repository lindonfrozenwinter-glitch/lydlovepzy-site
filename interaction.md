# 情侣纪念网站交互设计文档

## 核心交互功能

### 1. 登录系统
- **用户验证**：支持两个用户登录
  - 林粤冬（用户名：林粤冬，密码：2006111620250601）
  - 蒲振宇（用户名：蒲振宇，密码：2008062220250601）
- **背景效果**：登录页面背景虚化，营造浪漫氛围
- **色调切换**：登录页面支持5种配色方案切换
- **申请功能**：提供"我要申请"选项，可发送邮件申请访问权限

### 2. 写日常功能
- **身份绑定**：
  - 林粤冬只能以"狮王"身份发布日常
  - 蒲振宇只能以"小鱼"身份发布日常
- **权限控制**：防止冒用对方身份发布，提示"小坏蛋，不准偷偷替对方发日常嗷～"
- **内容限制**：主题16字以内，正文2000字以内
- **发布效果**：发布后直接显示在网页上，带有Q弹动效

### 3. 日常管理
- **删除权限**：
  - 林粤冬可以删除狮王发布的日常
  - 蒲振宇可以删除小鱼发布的日常
- **错误提示**：无权限删除时显示"不要删除对方的日常嗷～"
- **初始内容**：内置一条狮王发布的初始日常

### 4. 照片墙功能
- **上传系统**：支持多张照片上传
- **爱心汇聚**：照片达到一定数量自动汇聚成爱心形状
- **交互效果**：鼠标悬停时照片散开，便于查看
- **液态玻璃**：所有界面采用iOS 26风格的液态玻璃效果

### 5. 我们一起做
- **任务清单**：200个不重复的情侣活动任务
- **完成状态**：
  - 灰色：未完成
  - 半色：一方完成
  - 全色：双方都完成
- **动效反馈**：双方都完成时弹出"恭喜🎉，我们又一起做了一件事！"
- **进度追踪**：实时显示已完成任务数量

### 6. 一键报备
- **位置获取**：读取用户当前位置信息
- **自动发布**：以日常形式自动发布位置信息
- **格式规范**：
  - 主题：报备位置
  - 内容：大致位置 + 精确经纬度
- **权限处理**：无法获取位置时给出友好提示
- **不可删除**：报备产生的日常任何情况下都不可删除

### 7. 个人设置
- **密码修改**：用户可以修改自己的密码
- **验证机制**：修改前需输入原密码验证
- **色调切换**：支持5种配色方案实时切换
- **全局应用**：色调改变影响所有页面，包括登录页

## 交互流程设计

### 登录流程
1. 用户进入登录页面
2. 选择喜欢的色调（可选）
3. 输入用户名和密码
4. 验证通过后进入主页面
5. 支持"我要申请"功能发送邮件

### 写日常流程
1. 点击"写日常"按钮
2. 系统自动识别用户身份
3. 选择对应的发送人（狮王/小鱼）
4. 填写主题和正文
5. 点击发布，带有Q弹动效显示

### 照片上传流程
1. 点击"选择文件"按钮
2. 选择照片文件
3. 自动上传到照片墙
4. 达到数量后自动形成爱心形状
5. 鼠标悬停可查看单张照片

### 任务完成流程
1. 进入"我们一起做"页面
2. 浏览200个情侣任务
3. 点击任务进行标记
4. 双方都完成后显示庆祝动效
5. 实时更新完成进度

## 动效设计

### Q弹动效
- 按钮点击时的弹性反馈
- 卡片出现的弹跳效果
- 页面切换的流畅过渡

### 液态玻璃效果
- 所有卡片和容器采用半透明玻璃质感
- 背景模糊营造深度感
- 边缘发光增强视觉效果

### 交互反馈
- 鼠标悬停时的微妙动画
- 操作成功的庆祝效果
- 错误提示的温和提醒

## 响应式设计
- 支持桌面端和移动端
- 自适应布局确保良好体验
- 触摸友好的交互元素
